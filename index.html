一键开启BBR

1、自动，选BBR+FQ

wget --no-check-certificate -O tcpx.sh https://raw.githubusercontent.com/ylx2016/Linux-NetSpeed/master/tcpx.sh

chmod +x tcpx.sh

./tcpx.sh

2、手动开启BBR

echo "net.core.default_qdisc=fq" >> /etc/sysctl.conf

echo "net.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf

输入以下命令，保存并生效

sysctl -p

docker合集

docker安装

debian、ubuntu
curl -fsSL https://test.docker.com -o test-docker.sh sudo sh test-docker.sh
或者
wget -qO- get.docker.com | bash
设置开机自启
systemctl enable docker # 设置开机自动启动
测试是否安装成功：
docker -v
以下配置会增加一段自定义内网 IPv6 地址，开启容器的 IPv6 功能，以及限制日志文件大小，防止 Docker 日志塞满硬盘：

{
    "log-driver": "json-file",
    "log-opts": {
        "max-size": "20m",
        "max-file": "3"
    },
    "ipv6": true,
    "fixed-cidr-v6": "fd00:dead:beef:c0::/80",
    "experimental":true,
    "ip6tables":true
}
EOF

然后重启 Docker 服务：
systemctl restart docker

docker-compose安装

下载 Docker Compose 的当前稳定版本
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.3/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

将可执行权限应用于二进制文件：
sudo chmod +x /usr/local/bin/docker-compose

创建软链：
sudo ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose

测试是否安装成功：
docker-compose version

docker项目-Nginx Proxy Manager

创建安装目录

sudo -i

mkdir -p /root/data/docker/npm

cd /root/data/docker/npm

在文件夹下nano一个docker-comepose.yml
nano docker-compose.yml

粘贴以下内容，再按CTRL+X，按Y，按回车，保存退出

version: '3'
services:
  app:
    image: 'jc21/nginx-proxy-manager:latest'
    restart: unless-stopped
    ports:
      - '80:80'  #冒号左边可以改成自己服务器未被占用的端口
      - '81:81'  #冒号左边可以改成自己服务器未被占用的端口
      - '443:443' #冒号左边可以改成自己服务器未被占用的端口
    volumes:
      - ./data:/data # 冒号左边可以改路径现在是表示把数据存放在在当前文件夹下的 data 文件夹中
      - ./letsencrypt:/etc/letsencrypt  #冒号左边可以改路径，现在是表示把数据存放在在当前文件夹下的 letsencrypt 文件夹中

docker项目-碎片化知识管理Memos

创建安全目录

sudo -i

mkdir -p /root/data/docker/memos

cd /root/data/docker/memos

运行
docker run -d --name memos -p 5230:5230 -v /root/data/docker/memos/.memos/:/var/opt/memos neosmemo/memos:latest

或者使用compose安装
在文件夹下nano一个docker-comepose.yml
nano docker-compose.yml

粘贴以下内容，再按CTRL+X，按Y，按回车，保存退出

version: "3"
services:
  memos:
    image: neosmemo/memos:latest
    container_name: memeos
    hostname: memeos
    ports:
      - "5230:5230"
    volumes:
      - /root/data/docker/memos/.memos/:/var/opt/memos
    restart: always

运行
docker-compose up -d

vitepress合集

环境搭建

npm包管理器初始化

npm init

安装vitepress

npm add install -D vitepress

安装向导

一路按回车键即可，全默认
npx vitepress init
询问Where should VitePress initialize the config?输入docs,如下

 Where should VitePress initialize the config?
│./docs

配置站点名称和描述

文件位置.vitepress/config.js
文件内容

// .vitepress/config.js
export default {
  // site-level options
  title: '网站名称',
  description: '描述和关键字',

  themeConfig: {
    // theme-level options
  }
}

配置package.json

加入以下内容一边快速运行

"scripts": {
    "docs:dev": "vitepress dev docs",
    "docs:build": "vitepress build docs",
    "docs:preview": "vitepress preview docs"
  },

写个hello word

echo '# Hello VuePress' > docs/index.md

运行命令

以下2选1，效果是一样的

npm run docs:dev
npx vitepress dev docs

查看效果

运行结果,会展示一个网址，这就是网站的本地网址

http://localhost:5173

2、markdown文档实现侧边栏

在vscode里安装Markdown Preview Enhanced插件
用vscode打开 xxx.md 右键选择MPE:打开侧边预览

将光标放MD文档的第一行，然后按Ctrl+Shift+P（MacOS：command+shift+p）呼出命令面板，输入 Markdown Preview Enhanced: Create Toc 回车，会在光标位置生成一段代码

在md文档第一行加入

---
html:
    toc: true
    number_sections: true
    toc_depth: 6
    toc_float: true
    collapsed: true
    smooth_scroll: true
---

最终效果如图：

在设置中勾选Enable Script Execution，确保开启

右键选择MPE:打开侧边预览,在预览栏选择html导出，打开导出的html即可
